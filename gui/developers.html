Developer Guide

Everything you need to self-host Sigil, connect it to your Discord server, and extend it.

🔗 github.com/ShadowWalkerNC/Sigil

Overview

Sigil is a Node.js monorepo. start.js spawns two child processes — the Discord bot (src/index.js) and the GUI web server (gui/gui-server.js). Both run in the same Railway deployment.

  • Bot — discord.js v14, slash commands, automation events (welcome, goodbye, milestone, boost)
  • GUI server — Express.js, canvas image rendering via node-canvas, serves HTML pages
  • Database — SQLite via better-sqlite3 for per-guild automation config

Requirements

  • Node.js 18+
  • npm 9+
  • A Discord application with a bot token — discord.com/developers/applications
  • Bot scopes: bot, applications.commands
  • Bot permissions: Send Messages, Attach Files, Embed Links, Read Message History, View Channels
  • Privileged intents enabled in the dev portal: Server Members Intent, Presence Intent

Quick Start

1. Clone the repo

git clone https://github.com/ShadowWalkerNC/Sigil.git
cd Sigil

2. Install dependencies

npm install

3. Configure environment

cp .env.example .env
# Edit .env with your values (see Environment Variables below)

4. Register slash commands

node src/deploy-commands.js

This registers all slash commands globally. Allow up to 1 hour for Discord to propagate them.

5. Start Sigil

npm start
# or
node start.js

This starts both the bot and the GUI server. The GUI is available at http://localhost:8080 by default.

Railway Recommended

Railway is the easiest way to deploy Sigil. One-click deploy from GitHub, free tier available.

  1. Fork github.com/ShadowWalkerNC/Sigil
  2. Create a new Railway project → Deploy from GitHub repo → select your fork
  3. Go to Variables and add all required env vars (see below)
  4. Railway auto-detects Node.js and runs npm start
  5. Your GUI will be live at the Railway-provided URL immediately after deploy

The PORT variable is automatically set by Railway — no need to configure it manually.

Self-Hosted (VPS / Local)

Using PM2 (recommended for production)

npm install -g pm2
pm2 start start.js --name sigil
pm2 save
pm2 startup

Using a reverse proxy (nginx)

server {
    listen 80;
    server_name yourdomain.com;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}

Docker Community

A Dockerfile is not included in the repo yet, but you can create one:

FROM node:20-slim
WORKDIR /app
RUN apt-get update && apt-get install -y \
    libcairo2-dev libpango1.0-dev libjpeg-dev libgif-dev librsvg2-dev
COPY package*.json ./
RUN npm ci --omit=dev
COPY . .
EXPOSE 8080
CMD ["node", "start.js"]
docker build -t sigil .
docker run -d --env-file .env -p 8080:8080 sigil
